Yarn构建工具入门

发表于 2017-02-23   |   分类于 tools   |  
Yarn是一个facebook新出的包管理工具，在这之前，我们一直都是使用npm，或者bower来进行包管理。那明明npm和bower用着好好的，为啥要出一个新的yarn。
打开yarn官网，我去醒目的打字跃然眼前；

快速、可靠、安全的依赖管理。

说实话，npm在没有科技工具的情况下，安装速度的确令人难受，不过我们有了cnpm后，其实也就还好了。毕竟十分钟一次的更新，和npm使用起来没啥差别。不多说，首先看他极力推崇的自身的三大特性
  1. 极速。
    2. Yarn 缓存它下载的每个包，所以无需重复下载。它还并行化操作以最大化资源利用，所以安装时间比以往快。（敢问如果版本不同咋办，当然，大部分情况下版本迭代没有那么快）
  2. 超级安全。
    Yarn 在每个安装包的代码执行前使用校验码验证包的完整性。（不明觉厉，貌似这么久，尚未发现下载下来的包出现不完整的状况。）
  3. 超级可靠。
    Yarn 使用一个格式详尽但简洁的 lockfile 和一个精确的算法来安装，能够保证在一个系统上的运行的安装过程也会以同样的方式运行在其他系统上。（完全不明白什么意思，是说跨平台能力很强吗）

除了这个三大重要特性外，yarn还列举以下特性

  1. 离线模式
    如果你以前安装了一个包，你可以没有网络连接再次安装它。
  2. 确定性
    不管是什么顺序，在不同的机器上的依赖会以同一方式安装。
  3. 网络性能
    Yarn 高效的排队请求并且避免请求瀑布以便最大化网络利用率。
  4. 相同的软件包
    从 npm 安装软件包并保持包处理流程相同。
  5. 网络适应（这个厉害，很实用）
    单个请求失败不会导致安装失败，请求失败时会重试。
  6. 扁平模式（这个貌似很强大的样子）
    解析不匹配的依赖版本为一个单一的版本来避导致免多个版本。
下面首先开始安装yarn，根据官网，它提供了这么几种安装方式
  1. 直接下载yarn安装包安装（感觉有点low逼的样子）
  2. 使用choco安装（Chocolatey 是一个 Windows 包管理器，尼玛还要再安装一个什么Chocolatey，有点麻烦）

==其实直接用npm安装也可以，为啥官网不说呢，大家都懂的==

安装完成后就是初步的使用了
不知道为啥，我的yarn init直接在gitbash里面不好使，还得打开cmd才行，和用bower的时候一个鬼样子.
到目前为止，基本上已经算是yarn的基础使用入门了
